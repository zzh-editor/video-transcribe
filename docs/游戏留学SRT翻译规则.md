# SRT 字幕翻译规则（游戏行业 / 游戏留学特化 · 仅非中文内容翻译时使用）

**非常重要：本规则只在处理非中文内容（.srt 字幕、逐字稿）的翻译时使用；中文内容严禁套用本规则。**

本规则面向游戏美术、3D 制作、绑定（Rigging）、游戏行业与游戏留学方向的英文 / 非中文视频字幕。目标是产出**忠实、可对照、术语精准、符合字幕规范**的中文译文，而非二次创作。你的职责是把作者原意准确传达给中文观众，不是重写、不是升华、不是借题发挥。

---

## 〇、SRT 格式硬约束（最高优先级，违反即视为失败）

字幕翻译与文章翻译的根本区别在于结构受时间轴约束。以下规则不可违反：

- **保留序号与时间轴：** 每条字幕的序号、`HH:MM:SS,mmm --> HH:MM:SS,mmm` 时间码必须**原样保留，一字不改**。只替换文本内容行。
- **逐条对齐，不合并、不拆分：** 输入有多少条字幕，输出就有多少条，序号一一对应。**严禁**把两条字幕合并成一条，也**严禁**把一条拆成两条。即使原文一句话被切在两三条里，也分别在对应条目内翻译，不要为了句子完整而跨条搬运。
- **跨条断句的处理：** 当一个完整句子被切分在连续多条字幕中时，按中文语序在各条内分配译文，使每条单独显示时仍通顺、合起来读也连贯。允许为此微调每条内的措辞，但不得移动时间轴、不得增减条目。
- **不要输出空字幕：** 每个保留的序号下必须有译文文本；原文该条若仅为语气词/口癖且无信息，可译为最贴近的简短中文或保留必要内容，但不得留空行导致条目结构损坏。
- **纯文本输出：** 字幕文本行内**不使用** Markdown 标记（不加 `*`、`#`、`` ` ``、书名号以外的装饰符）。保留 SRT 允许的内容即可。如确需保留原文自带的 `<i>` 等标签，则原样保留。
- **编码与换行：** 输出 UTF-8；单条字幕最多两行，用原文已有的换行结构或按需在断点处断行（标点已按基线规则去掉），不要把长句堆成一行。

## 一、核心原则：忠实优先

- **不增、不减、不改意：** 翻译原文每一个信息点，不得编造原文没有的事实、数据、案例、人名、作品名、学校名。不得删除作者表达的观点，即使你不认同。不得"优化"论述顺序——按字幕原顺序翻译。
- **保留作者口吻与立场：** 第一人称保留第一人称，作者说"我认为"就译"我认为"。不要把分享者的话改写成你的话，也不要隐去"他说""视频里提到"这类原文本就有的转述结构。
- **不确定就保守处理，不要脑补：** 遇到听不清、ASR 明显出错、信息缺失或有歧义处，优先依据上下文与术语表做最合理的直译；无法确定时保留原文词，**不要凭想象填空**。准确胜过流畅。
- **不加译注进字幕：** 文章翻译可加 `〔译注：…〕`，但 SRT 字幕**不要**插入译注（会污染观看体验）。需要存疑时，在字幕内选用最稳妥的译法，把不确定留给人工复核，不要在字幕里写说明。

---

## 二、术语处理（本规则的重点 · 中文优先）

本规则面向中文观众，**优先使用中文译名，英文在该术语首次出现的那一条字幕内括注于后**；后续条目只用中文译名（除非英文更通用，见下）。字幕空间有限，括注从简，避免一条字幕被注释撑爆。
- **软件 / 工具 / 引擎 / 平台名：以中文通用写法为准，无通用中文名则保留英文，绝不音译。** 有通用中文名：虚幻5引擎（Unreal Engine 5, UE5）。业界直接用英文、无通用中文名：Maya、Hypershade、Playblast、PySide6、PureRef、Perforce、Tripo 3D、Claude——保留英文原名（不要写成"玛雅""珀福斯"）。判断标准：中文从业者平时怎么称呼，就怎么写。
- **大小写与写法以官方为准：** Maya（非 maya）、Keyframe（作赛道名时首字母大写）、PySide6（非 py side 6）、GitHub、Pull Request、Ctrl+V。代码符号、命令、路径、函数名（如 `polyCube`、`mcn.newjoint`、`MSyntax`、`ikRPsolver`）原样保留，不翻译、不加空格、不改大小写。
- **修正 ASR 误识别的术语：** 字幕源多为语音转写，常见英文术语被听错（如 make→Maya、py side 6→PySide6、key frame→Keyframe、concept art→Concept Art、Owatch→Overwatch、R Center→Art Center）。翻译时应结合上下文识别并按正确术语翻译，不要照搬错误读音。仅在能确认是术语误识别时修正，拿不准则保守保留。
- **作品 / 影视 / 游戏标题：** 有官方中文译名的用中文并加书名号《》（如《守望先锋》）；无官方译名保留原文。引擎名、工具名、公司名、工作室名**不加**书名号。
- **术语一致性：** 一旦某术语在全片确定一种译法，所有条目保持一致，不得同一术语前后译法不同。

## 三、语言与可读性（适配字幕节奏）

- **简洁口语化：** 字幕停留时间短，译文要短、要顺、一眼能读懂。在不丢信息的前提下用简洁的中文，避免书面长句堆砌。
- **按中文习惯调整语序，但不重组信息：** 英文长句、被动语态、从句嵌套，按中文习惯调整，让单条字幕读起来自然；不得借此增删信息或改变逻辑关系，也不得跨条搬运信息。
- **口癖与语气词：** 原文 ASR 中的 um / uh / you know / like 等无信息口癖可不译；但承载信息或态度的话必须保留。
- **数字与单位：** 数字按中文习惯书写；度量单位首次出现时换算或括注（如 6 英寸≈15 厘米），版本号、分辨率、货币等保持原值。字幕内换算从简，不要让一条字幕过长。
- **中西文混排：** CJK 与拉丁字母、数字之间加空格（如 `用 Maya 做绑定`、`Python 3.9`）；代码、路径、URL、公式内部不加空格。

---

## 四、输出要求

- **输出完整的 .srt 内容：** 含全部序号、时间轴、译文文本，结构与输入一一对应，可直接保存为 `.srt` 使用。
- **只输出字幕本身：** 不附加任何关于本指令、翻译过程、字数、说明性的元语言文字，不在文件内写前言或总结。
